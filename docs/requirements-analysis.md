# 校园问卷投票系统需求分析

## 1. 项目目标

本项目是一个基于 JSP + Servlet + JDBC + PostgreSQL 的校园问卷投票系统。

系统面向校园内普通用户，提供注册、登录、发布投票、查看投票列表、提交投票、查看统计结果、删除投票等功能。项目采用完整 MVC 架构：

- Model：实体类、DAO、数据库访问工具类。
- View：JSP 页面负责展示表单、列表、结果和提示信息。
- Controller：Servlet 负责接收请求、调用业务逻辑、跳转 JSP。

本阶段只做需求分析，不进入编码实现。

## 2. 使用者与权限

系统暂定只有一种用户角色：普通用户。

普通用户登录后可以：

- 发布新的投票问卷。
- 查看所有问卷列表。
- 对未投过的问卷进行投票。
- 查看问卷投票结果。
- 删除自己发布的问卷。

未登录用户可以：

- 访问登录、注册页面。
- 提交登录、注册请求。

未登录用户不可以：

- 发布问卷。
- 提交投票。
- 删除问卷。

## 3. 核心页面需求

### 3.1 登录注册页面

页面名称建议：`login.jsp`、`register.jsp`

功能：

- 用户输入用户名和密码进行登录。
- 新用户输入用户名、密码进行注册。
- 登录成功后进入问卷列表页面。
- 注册成功后跳转到登录页面或直接进入问卷列表页面。
- 登录失败、注册失败时在页面展示错误提示。

Cookie 使用：

- 可以使用 Cookie 保存最近一次登录的用户名，方便下次自动填充用户名输入框。
- 不保存明文密码。

### 3.2 问卷发布页面

页面名称建议：`vote_publish.jsp`

功能：

- 登录用户填写问卷标题。
- 登录用户填写投票选项。
- 至少需要 2 个选项。
- 发布成功后跳转到问卷列表页面。
- 发布失败时展示错误提示。

需求边界：

- 根据当前指定实体 `VoteQuestion`、`VoteOption`，一个问卷暂定只包含一个投票题目和多个选项。
- 暂不实现复杂问卷，例如一个问卷包含多道题、分页题目、题型配置等。

### 3.3 问卷列表页面

页面名称建议：`vote_list.jsp`

功能：

- 展示所有已发布问卷。
- 展示问卷标题、发布者、发布时间。
- 提供“去投票”入口。
- 提供“查看结果”入口。
- 对当前登录用户自己发布的问卷，提供“删除”入口。

### 3.4 投票提交页面

页面名称建议：`vote_submit.jsp`

功能：

- 展示问卷标题和所有选项。
- 用户选择一个选项并提交。
- 提交成功后跳转到结果页面。
- 未选择选项时展示提示。
- 已投过该问卷时禁止重复提交，并提示用户查看结果。

Session 使用：

- 登录成功后，Session 中保存当前登录用户信息。
- 用户投票成功后，Session 中记录该用户在当前浏览器会话内已经投过的问卷 ID。
- 后续再次访问同一问卷投票提交页面时，Servlet 根据 Session 判断是否已经投票。

说明：

- Session 防重复投票只对当前浏览器会话有效。
- 如果需要跨浏览器、跨设备、退出后仍严格禁止重复投票，后续设计阶段需要增加投票记录表。本阶段先按课程要求使用 Session 实现。

### 3.5 投票结果页面

页面名称建议：`vote_result.jsp`

功能：

- 展示问卷标题。
- 展示每个选项的票数。
- 展示每个选项的占比。
- 展示总票数。
- 可以返回问卷列表。

统计方式：

- 每个选项保存自己的票数。
- 查询结果时汇总所有选项票数得到总票数。
- 占比 = 当前选项票数 / 总票数。
- 总票数为 0 时，占比显示为 0%。

### 3.6 问卷删除页面

页面名称建议：`vote_delete.jsp`

功能：

- 当前登录用户只能删除自己发布的问卷。
- 删除前展示确认页面，避免误删。
- 确认删除后删除问卷及其选项。
- 删除成功后返回问卷列表页面。
- 删除不存在、无权限删除时展示错误提示。

## 4. 实体类需求

### 4.1 User

字段建议：

- `id`：用户 ID，数据库主键。
- `username`：用户名，唯一。
- `password`：密码。
- `createdAt`：注册时间。

说明：

- 密码使用 bcrypt 哈希存储，即使数据库泄露也无法反推出原始密码。
- 用户名不能为空，不能重复。

### 4.2 VoteQuestion

字段建议：

- `id`：问卷 ID，数据库主键。
- `title`：问卷标题。
- `userId`：发布者用户 ID。
- `createdAt`：发布时间。

说明：

- `VoteQuestion` 表示一个投票题目。
- 删除问卷时，需要同步删除它下面的所有选项。

### 4.3 VoteOption

字段建议：

- `id`：选项 ID，数据库主键。
- `questionId`：所属问卷 ID。
- `content`：选项内容。
- `voteCount`：当前票数。

说明：

- 一个 `VoteQuestion` 至少包含两个 `VoteOption`。
- 用户投票时，对被选中的 `VoteOption.voteCount` 加 1。

## 5. Servlet 需求

### 5.1 UserServlet

建议路径：

- `/user/login`
- `/user/register`
- `/user/logout`

职责：

- 处理登录。
- 处理注册。
- 处理退出登录。
- 登录成功后写入 Session。
- 登录成功后写入最近登录用户名 Cookie。

### 5.2 VotePublishServlet

建议路径：`/vote/publish`

职责：

- GET：进入问卷发布页面。
- POST：接收标题和选项，保存问卷和选项。
- 只允许登录用户访问。

### 5.3 VoteListServlet

建议路径：`/vote/list`

职责：

- 查询所有问卷。
- 转发到问卷列表 JSP。
- 只允许登录用户访问。

### 5.4 VoteSubmitServlet

建议路径：`/vote/submit`

职责：

- GET：展示某个问卷的投票表单。
- POST：接收用户选择的选项，完成投票。
- 使用 Session 判断当前会话是否已经投过该问卷。
- 投票成功后跳转到结果页面。

### 5.5 VoteResultServlet

建议路径：`/vote/result`

职责：

- 根据问卷 ID 查询问卷和选项票数。
- 计算总票数和占比。
- 转发到结果 JSP。

### 5.6 VoteDeleteServlet

建议路径：`/vote/delete`

职责：

- GET：进入删除确认页面。
- POST：执行删除。
- 只允许问卷发布者删除自己的问卷。

## 6. 数据存储需求

数据库：PostgreSQL。

建议表：

- `users`
- `vote_questions`
- `vote_options`

表关系：

- 一个用户可以发布多个问卷。
- 一个问卷属于一个用户。
- 一个问卷包含多个选项。
- 一个选项属于一个问卷。

后续编码阶段需要确认：

- 是否允许在 `pom.xml` 中新增 PostgreSQL JDBC 驱动依赖。
- 数据库连接信息使用配置文件还是先写在工具类常量中。

## 7. MVC 分层需求

建议包结构：

```text
org.test.model       实体类
org.test.dao         数据库访问类
org.test.servlet     Servlet 控制器
org.test.util        JDBC 工具类
```

建议 JSP 目录：

```text
src/main/webapp/
  login.jsp
  register.jsp
  vote_publish.jsp
  vote_list.jsp
  vote_submit.jsp
  vote_result.jsp
  vote_delete.jsp
```

分层规则：

- JSP 不直接连接数据库。
- Servlet 不直接拼接复杂 SQL，数据库操作交给 DAO。
- DAO 不处理页面跳转。
- 实体类只保存数据，不处理请求。

## 8. 关键业务流程

### 8.1 注册流程

1. 用户进入注册页面。
2. 输入用户名和密码。
3. `UserServlet` 检查用户名是否重复。
4. 不重复则保存用户。
5. 注册成功后跳转登录页面。

### 8.2 登录流程

1. 用户进入登录页面。
2. 输入用户名和密码。
3. `UserServlet` 校验用户信息。
4. 校验成功后把 `User` 放入 Session。
5. 写入最近登录用户名 Cookie。
6. 跳转问卷列表页面。

### 8.3 发布问卷流程

1. 用户登录后进入发布页面。
2. 填写标题和至少两个选项。
3. `VotePublishServlet` 校验输入。
4. 保存问卷。
5. 保存问卷选项。
6. 跳转问卷列表页面。

### 8.4 投票流程

1. 用户从列表进入投票页面。
2. `VoteSubmitServlet` 查询问卷和选项。
3. 检查 Session 中是否已经投过。
4. 未投过则允许提交。
5. 提交后对应选项票数加 1。
6. 把问卷 ID 写入 Session 已投票集合。
7. 跳转投票结果页面。

### 8.5 查看结果流程

1. 用户从列表或投票成功页进入结果页。
2. `VoteResultServlet` 查询问卷和选项。
3. 统计总票数和百分比。
4. JSP 展示结果。

### 8.6 删除问卷流程

1. 用户从列表点击删除。
2. `VoteDeleteServlet` 校验登录状态和发布者权限。
3. GET 请求展示删除确认页面。
4. POST 请求执行删除。
5. 删除选项和问卷。
6. 返回问卷列表。

## 9. 输入校验与异常提示

需要校验：

- 用户名不能为空。
- 密码不能为空。
- 注册用户名不能重复。
- 问卷标题不能为空。
- 问卷选项不能为空。
- 问卷至少有两个有效选项。
- 投票时必须选择一个选项。
- 问卷 ID、选项 ID 必须合法。
- 删除问卷必须是发布者本人。

提示方式：

- Servlet 将错误信息放入 request。
- JSP 在页面顶部或表单附近显示错误信息。

## 10. 验收标准

需求分析通过后，后续实现需要满足：

- 可以完成注册、登录、退出。
- 登录状态使用 Session 保存。
- 最近登录用户名可以通过 Cookie 回显。
- 登录后可以发布问卷。
- 问卷列表可以展示所有问卷。
- 用户可以进入问卷投票页面并提交投票。
- 同一 Session 内不能对同一问卷重复投票。
- 投票结果页面可以正确显示票数、总票数和百分比。
- 问卷发布者可以删除自己的问卷。
- 非发布者不能删除别人的问卷。
- 项目保持 MVC 分层，JSP 不直接访问数据库。
- 项目可通过 Maven 编译打包。
